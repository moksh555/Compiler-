# ⚙️ Compiler — ActionScript Subset Compiler

A full **compiler front-end** for a statically-typed, class-based language (ActionScript-inspired syntax, `.as` files). Built from scratch using **JFlex** for lexical analysis and **CUP** for LALR(1) parser generation. The compiler performs lexing, parsing, and semantic analysis including type checking and symbol table management.

## ✨ Features

- **Lexer (JFlex-generated)** — tokenizes keywords (`class`, `void`, `final`, `return`, `if`, `else`, `while`, `print`, `read`), primitive types (`int`, `float`, `bool`, `char`, `string`), operators, literals, and identifiers
- **LALR(1) Parser (CUP-generated)** — grammar covers class declarations, field/method declarations, typed argument lists, statements (if/else, while, assignment, return), and expressions with full operator precedence
- **Type system with coercion** — supports `int`, `float`, `bool`, `char`, `string`, `void`; allows numeric widening (int → float, bool → int) tracked via `SuperToken.isCoercible()`
- **Symbol table & semantic analysis** — validates variable declarations, detects redefinitions (`redefVar`, `redefMethod`, `redefVarAsMethod`), enforces return type correctness (`noReturn`, `returnTypeBad`), and checks function call existence (`callNonExistFunc`)
- **Error test suite** — includes 15+ `.as` test programs covering invalid scenarios (bad decrement/increment, incompatible binary ops, illegal type casts, reassignment of finals, ternary type mismatch)
- **Build automation** — `Makefile` orchestrates JFlex + CUP code generation and `javac` compilation

## 🚀 Setup & Installation

**Prerequisites:** Java 11+, GNU Make

```bash
# 1. Clone the repository
git clone https://github.com/moksh555/Compiler-.git
cd Compiler-

# 2. Build the compiler (runs JFlex, CUP, then javac)
make

# 3. Run the compiler against a test file
java parser < fullValidProgram.as

# 4. Run against an error-case file
java parser < badDec.as
```

## 🏗️ Architecture & Design

The compiler is structured as a classic three-phase front-end:

1. **Lexical Analysis** (`Lexer.java`) — DFA-based scanner generated by JFlex from `tokens.jflex`. Produces typed tokens consumed by the parser.
2. **Syntactic Analysis** (`parser.java`) — LALR(1) parser generated by CUP from `grammar.cup`. Builds a typed AST using Java classes for each non-terminal (`Program`, `Methoddecl`, `Stmt`, `Expr`, `BinaryOp`, etc.).
3. **Semantic Analysis** (`SuperToken.java` + AST nodes) — Each AST node performs its own semantic checks during tree construction. A shared static `SymbolTable` (via `SuperToken`) tracks declared variables and functions. The `CurrentFunction` inner class tracks the active function's return type to validate return statements.

### Operator Precedence (excerpt from grammar)
```
right  NOT, UPLUS, UMINUS, INCREMENT, DECREMENT
left   MULTIPLY, DIVISION
left   PLUS, MINUS
left   LTS, GTS, LTE, GTE
left   EQ, NE
left   AND → OR → ASSIGNMENT → QUESTION (ternary)
```
